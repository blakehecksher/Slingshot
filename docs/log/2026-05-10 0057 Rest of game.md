## What worked

- Sharing the existing `sampleGravityAt` between the player, enemies, and projectiles meant projectile curving "just works" without any new physics code.
- Collapsing the originally-planned UPGRADE_DEFS into `BuiltinPartDef.stats` removed an entire reconciliation layer in the hangar.
- Keeping `Ship` synchronous and giving it a `setVisual(BuiltShip)` swap kept the resolver and the GLB loading complexity out of the hot physics path.

## What didn't and why

- AI-generated GLBs in `public/ships/` were intentionally skipped per user direction; the pipeline supports them via `manifest.fullModel`, but no manifest references them yet.
- No live in-browser playtest yet — the user runs the dev server themselves; per memory, no commits are made during iterative dev.

## Session continuation — polish pass

After initial implementation verified and running, additional fixes and features applied:

### Flight tuning fix
- `FORWARD_THRUST: 200`, `REVERSE_THRUST: 65`, `FORWARD_THRUST_BIAS: 1.0`. Previously reverse was stronger than forward; ratioed correctly now.
- `BRAKE_DAMPING: 1.4`, `SPEED_ASSIST_DAMPING: 0.6` for feel.

### Hangar gamepad double-fire fix
- `HangarUI` was handling `b3` (Y) internally; main input loop also edge-detected `toggleHangar` on same frame → double-fire on open/close.
- Fix: `HangarUI` ignores `b3` entirely; only main loop handles `toggleHangar`.

### Enemy visibility overhaul
- Enemies spawning but invisible due to spawn z-range only in deep field + COL_ASTEROID collision causing launch from inside rocks.
- Fixed: spherical-shell spawn with 12-attempt rejection sampling requiring ≥90m clearance from all asteroids. Removed COL_ASTEROID from enemy collision filter (ghosts through rocks). Scaled to 1.6×.
- Visual boost: emissive cockpit `emissiveIntensity: 2.4, toneMapped: false`, additive halo sphere mesh, point light strength 4.2 range 60 m.
- `ENEMY_TUNING.COUNT: 32` — now tunable. Lazy-fill in `update()` keeps count at target without discrete respawn batches.

### Projectile speed fix
- Was 180 m/s (player could outrun). Light cutter → 720 m/s, heavy slug → 480 m/s, enemy → 560 m/s.

### Asteroid field overhaul
- Switched to uniform spherical-shell distribution (cube-root sampling). Dropped hand-placed rocks.
- `SPHERE_INNER: 520` — 520 m clear bubble around base so player doesn't spawn into a boulder.
- `SIZE_INNER_MAX: 0.32` — size bias: inner-shell rocks capped at 32% of RADIUS_RANGE; cap grows linearly to 1.0 at SPHERE_OUTER. Giant asteroids only appear in deep field.
- `COUNT: 900`, `SPHERE_OUTER: 8200`, `RADIUS_RANGE: 240`.
- Player-spawn-death bug caused by SIZE_INNER_MAX=1.0 (unset) + SPHERE_INNER=240; fixed by both knobs together.

### Skybox infinite feel
- Stars + dome now live in a `THREE.Group` whose `.position` is copied to `camera.position` each render frame.
- Parallax never reveals the sphere boundary; stars feel truly infinite.

### Lock-on targeting + lead reticle
- `pickLockTarget()` in `main.ts`: cone search 35°, range 1800 m, score = `dot − range * 0.25`. Triggered by R3 (b10/b11 depending on browser).
- `src/render/reticle.ts`: DOM crosshair (always center), red lockBox (at enemy screen position), gold lead circle (quadratic intercept point).
- `interceptTime()`: solves `(|relVel|² − μ²)t² + 2(relPos·relVel)t + |relPos|² = 0`; prefers smaller positive root.
- Lead disappears when no positive root (target fleeing faster than muzzle speed).

### First-person / third-person toggle
- `Back` button (b8) cycles camera between chase-3rd and cockpit-1st.
- Reticle hidden in 3rd-person; shown in 1st-person for aiming.

### Hangar two-mode gamepad nav
- `rows` mode: D-pad/stick up–down navigates slots; A enters options mode.
- `options` mode: any D-pad direction cycles parts with live 3D preview update; A confirms+returns, B reverts+returns.
- Gold border in options mode, cyan on focused slot in rows mode.
- Start (b9) → Apply; X (b2) → Reset.
